# Cybar-2 项目文件清理与分析报告

本报告列出了项目中识别出的不再使用、冗余或临时生成的文件，并对其作用进行了说明。

## 1. 废弃的后端核心文件
建议：归档或删除。这些文件已被 `server/` 目录下的模块化代码取代。

- **server.js**
  - **作用**: 旧版的单文件后端服务入口。
  - **现状**: 当前项目的 `package.json` 和启动脚本 (`npm start`) 指向 `server/index.js`。该文件包含大量旧代码，不再被系统加载。

- **pnpm-lock.yaml**
  - **作用**: pnpm 包管理器的依赖锁定文件。
  - **现状**: 项目当前使用 `package-lock.json` (npm)，该文件是旧的残留，可能导致依赖版本混淆。

## 2. 废弃的数据存储文件 (JSON)
建议：归档或删除。这些文件曾用于从 `server.js` 读取数据，现已被 MySQL 数据库取代。

- **users.json**
  - **作用**: 旧版用户数据存储。
- **comments.json**
  - **作用**: 旧版评论数据存储。
- **likes.json**
  - **作用**: 旧版点赞数据存储。
- **favorites.json**
  - **作用**: 旧版收藏数据存储。

## 3. 废弃的前端脚本
建议：删除。

- **script.js**
  - **作用**: 旧的主页脚本。
  - **现状**: 文件内容为空（仅包含注释），且未被任何 HTML 页面引用（`index.html` 使用的是 `js/theme.js` 等）。

## 4. 临时文件与构建产物
建议：删除。

- **AI_KEY_CONFIGURED_SUCCESS.md**
  - **作用**: 脚本自动生成的标记文件，用于指示 AI Key 配置成功。
  - **现状**: 仅作为状态标记，对项目运行无实际作用。

- **count.txt**
  - **作用**: 项目代码行数或其他统计信息的临时记录文件。

- **avatar-initial_user_1-1765789781034-77539492.jpg** (文件名可能略有不同)
  - **作用**: 用户头像文件。
  - **现状**: 该文件位于项目根目录，属于位置错误。正常的上传文件应位于 `uploads/avatars/` 目录中。

## 5. 数据库备份文件 (SQL)
建议：移动到 `sql/` 或 `backup/` 目录，或只保留最新版本。

- **cybar1.sql**
- **cybar2.sql**
  - **作用**: 数据库转储/备份文件。包含数据库结构和初始数据。
  - **现状**: 即使需要保留用于初始化，建议统一管理，而非散落在根目录。

## 6. 开发辅助与测试脚本 (非生产环境必需)
建议：保留在 `scripts/` 目录或保持现状，主要用于开发调试。

- **setup-api-key.bat / .ps1**: 用于快速配置环境变量的脚本。
- **test-api-key.js**: 用于测试 AI API Key 是否有效的独立脚本。
- **verify-ai-function.js**: 用于验证 AI 功能逻辑的独立脚本。
- **start-and-test.bat**: 用于启动服务并运行测试的批处理文件。
- **config.json.example**: 配置文件模板（应保留）。
